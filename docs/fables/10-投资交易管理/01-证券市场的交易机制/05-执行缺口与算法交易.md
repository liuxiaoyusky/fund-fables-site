# 五、执行缺口与算法交易

## 🏺 寓言故事 —— 《理想与现实的差距》

阿智计划以20元/股买入100股股票，但实际操作中没买到这么多、没买到这个价。实际收益和"理想收益"之间的差距，就叫**执行缺口**。

### 执行缺口的构成

执行缺口 = 显性成本 + 隐性成本，通常包括：

1. **显性成本**：佣金等直接费用（如0.2%）；
2. **延迟成本**：因为限价指令没成交，错过了价格上涨（如0.8%）；
3. **已实现损失**：实际成交价高于目标价（如1.2%）；
4. **机会成本**：计划买100股只买了80股，剩下的20股错过涨幅（如1%）。

总执行缺口 = 0.2% + 0.8% + 1.2% + 1% = **3.2%**。显性成本只占0.2%，隐性成本占了3%！

### 算法交易

为了减少执行缺口，现代交易使用**算法交易**——用计算机程序自动执行大额订单。

核心思路：把**大单拆成小单**，在不同时间逐步执行，减少对市场价格的冲击。

常见算法策略：
- **TWAP（时间加权平均价格）**：在固定时间间隔平均下单；
- **VWAP（成交量加权平均价格）**：根据市场成交量分布下单，跟随市场节奏；
- **TVOL（跟量算法）**：市场放量时多下，缩量时少下；
- **IS（执行缺口算法）**：目标是最小化执行缺口，快速以接近委托时的市场价格完成。

算法交易的优势：减少冲击成本、提高执行效率、避免人为情绪干扰、可测试优化策略。

---

**📖 原文定义**

> 执行缺口（Implementation Shortfall）是指实际交易结果与理想交易结果之间的差距，通常用来衡量交易执行的偏差程度。执行缺口反映了交易过程中产生的所有成本。

> 算法交易是指使用计算机算法来优化交易订单执行过程。这类算法通过遵循数量规则、用户指定的基准和约束条件，使用计算机来优化订单的执行路径、时间、价格以及数量。
