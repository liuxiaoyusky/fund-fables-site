# (4) 回购价格

## 💰 寓言故事 —— 《三百两该怎么还》

清河镇钱掌柜的染坊,三年前拉了县城孙掌柜入伙。孙掌柜掏了三百两,占了三成份额。契书末了一条写得明白——遇上"那一步",孙掌柜可以把份额"要回"。

可"要回"两个字好说,"按多少银子要回"才是真章。

这日两人又凑到一块儿,把这笔账翻来覆去地算。

孙掌柜先开口:"钱掌柜,当初我掏了三百两,这事儿你得认。"

钱掌柜点头:"认——三百两,一两不少。"

孙掌柜接着说:"三百两掏出去,放染坊里三年,日晒雨淋,柜上一文钱利息没给我。要是你今天把份额接回去,这三百两就得加一点'补偿'——我那三百两在你柜上躺了三年,得按'一年一成的利'来算。"

钱掌柜问:"一成,那三年下来,加多少?"

孙掌柜掰着指头:"三百两一年三十两利息,单算就是三十两一年,三年就是九十两;要是这九十两年年又算利,九年就滚成……"他摆摆手,"我先不和你算这个,单算简单,你也容易懂。"

钱掌柜点头:"那就单算——三百两加三年利息,共三百九十两。"

孙掌柜又说:"三百九十两是'染坊没出岔子'的算法。可要是哪一天真出了岔子——比如老几位里头谁偷了柜上的料子、做了假账——这种事儿,就不是三百九十两能了事的,得往上加,加到让人心里疼。"

钱掌柜皱眉:"那叫多少?"

孙掌柜一咬牙:"再加一截——三百九十两之外,再贴三成利息。"

钱掌柜苦笑:"这一截,就是给'不守规矩'的人加的'罚'。"

孙掌柜点头:"这是给'人出了岔'的算法——柜上能算清的,和'人心歪了'的,得两本账。"

两人议定,在契书最末一页把账本分了两栏:一栏叫"柜上的算法",一栏叫"人心的算法"。

钱掌柜叹口气:"这本账写进契书,谁也别想蒙混。"

孙掌柜也叹口气:"这本账写进契书,我那三百两才有着落。"

---

**📖 原文定义**

> 对于回购的价格,通常是投资方的投资成本加上以单利或复利计算的一定利息;也有少数情况是以投资方投资成本的一定倍数计算。正常情况下,单利或复利的区间一般在6%~10%较为常见。
>
> 实践中,也有区分不同的回购触发事件来区别设置回购利率的情况。比如,在"目标公司在交割后的一定年限内未能完成合格上市"这一触发事件时,计算一个相对较低的利率;但是在"创始团队或核心团队出现重大个人诚信问题"等关乎个人道德的情况下,计算一个相对较高带有一定惩罚性的利率。

**💡 对应点**

| 故事元素 | 概念对应 |
|---------|---------|
| 掏出去的银子加利息 | 投资成本加上一定利息 |
| 单算三年加九十两 | 单利计算 |
| 区间一般在一定范围内 | 单利或复利区间一般在6%~10% |
| 染坊没出岔子的算法 | "未能完成合格上市"等正常情形用较低利率 |
| 人心歪了往上加一截带罚的利率 | 重大个人诚信问题用较高且带惩罚性的利率 |
| 区别两本账 | 区分不同触发事件区别设置回购利率 |
