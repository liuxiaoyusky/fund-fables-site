# 4.1 产品的开发

## 故事

梁志明有天下午经过中环，看见英资保险行在门口贴了张招聘启事，写着"产品开发部——请英文佳、懂本地市场者"。他想了想自己没有一张像样的文凭作罢，但这件事倒让他留意起保险产品到底是怎么来的这个问题。

他问陈伯，陈伯在船公司做了三十年，说产品开发就是"看客人的口袋里有什么风险，再想一个办法把风险接过来"。这个说法朴素，但梁志明觉得已经说对了一半。

真正的产品开发，要做的事情更复杂。

首先是单个产品的开发。有一年市场突然流行起一种针对茶餐厅的保障计划，何师奶听同行说起，也想买一份。梁志明回去一查资料，发现这套产品最初就是公司里几个年轻伙计看到茶餐厅火灾新闻之后，向部门提议开发的。从提案到推出，前后不到三个月。结果两周之内，三家竞争对手就推出了类似的产品，条款差不多，保费还便宜一点。

"这就是为什么新产品要不断更新。"陈伯听完说，"旧产品一推出就被抄，抄完就便宜，抄完再改，抄完再便宜。"

其次是产品组合的开发。有一家做制衣的大客户来谈投保，厂要保、货要保、雇员要保，如果分开买，保费贵，管理也麻烦。保险行就设计了一个组合产品，把三项保障打包在一起，大客户一个签单全部搞定，比分开买便宜了两成。梁志明后来跟何师奶提起，何师奶想了想："打包卖，方便是方便，就是不知道万一出事哪一项赔哪一项。"

最后还有产品研究。梁志明说，研究分三块：一块看自己的产品有没有漏洞；一块看竞争对手在做什么，不能活在真空里；还要看整个市场在往哪个方向走。"比如这两年街坊开始问医疗险，以前根本没有这个需求，现在大家都开始存钱看病了。"

何师奶听到这里，点了点头："我就说以前在街坊里根本没人提，现在来吃饭的年轻人，十个有八个问医疗的事。"

陈伯把算盘珠子拨了一轮，说："产品开发，说到底就是两件事——知道别人要什么，再想一个办法在自己不亏的前提下给他。"

## 原文定义

保险公司产品开发的核心职责包括三个方面：个别产品的开发（根据市场需求持续更新）、产品组合的开发（将多种保障打包以吸引大型客户）以及产品研究（审视现有产品、观察竞争对手、分析市场趋势）。

## 对应点

| 故事元素 | 概念对应 |
|---|---|
| 茶餐厅保障产品从提案到推出不到三个月 | 个别产品开发：竞争激烈，生命周期短 |
| 三家竞争对手两周内推出类似产品 | 个别产品开发：容易被仿效和抄袭 |
| 制衣厂打包投保三项保障 | 产品组合开发：将多种保障组合吸引大客户 |
| 街坊开始问医疗险，梁志明注意到市场趋势 | 产品研究：市场趋势分析 |
| 何师奶想了解打包产品各项如何赔付 | 产品组合与消费者透明度需求 |
