# 5.1.2 保险的实务类别(Practical Classification of Insurance)

## 故事

周老板在上环做了三十年海味生意，那天他坐在梁志明对面，把一张手写的清单拍在桌上，说："我这么多货，进货的时候要走船运，库存放在仓库里，台风天还怕湿，店铺里客人来来去去也有风险。你们能不能给我一个说法，到底哪张单保哪样东西？"

梁志明把那几张保单翻出来，一张一张对着看，发现周老板的问题其实很合理。他的海味既有货物运输风险，又有仓库储存风险，还有对客人的责任风险——但这三块在公司的系统里，分属不同的"部门"，走了不同的流程。

张小姐那天正好从后台拿报表出来，看见两人对着文件发愁，顺手把报表放在桌上，说："周老板，我们公司内部是把业务分开的。运输那部分归海上保险部处理，仓库的财产归火险部，你店里客人如果出了事，那又是意外险那边。你问一张单保不保全部，当然不行，因为每个部门算账的方式都不一样。"

"那我怎么知道我到底买了什么？"周老板问。

梁志明想了想，说："周老板，这样吧，你告诉我你的货是从哪里来，到哪里去，平时放在哪里，谁来看店——我从我们公司的分法给你梳理一遍，你就清楚了。"

他拿了一张白纸，把周老板的生意拆成三块：货在船上时找谁，货在仓库时找谁，货在店里流转时找谁。张小姐在旁边补充："我们公司用的是英式分类法，火险、海上保险、意外险是三大块。但如果是美国那边的保险公司，分法又不一样——他们把人寿单独分出来，剩下的才再细分。"

周老板听着，忽然问："那你们怎么看我这张单是从哪里来的？"

梁志明笑了："这个就看谁替你办的了。你是通过我们保险行买的，这在内部算代理渠道；如果是你的经纪直接替你安排，那就是经纪渠道；如果你自己在报纸上看到广告打电话来，那就是直接渠道。三种来源，后台的管理方式也不一样。"

张小姐点头："包括记账方式、核保流程，都有区别。同样是火险，从不同渠道来的客人，我们内部跟踪的方式都不同。"

周老板把那页手写梳理收进口袋，说："我现在明白了。原来你们分门别类，不是为了为难客人，是为了自己管得清楚。"

梁志明送他出门时，周老板还说了一句："我回去还要给你们出一道题——我的海味铺，还有儿子的茶餐厅，能不能放在一起算？"

张小姐翻出报表，指给梁志明看："你看，这里就分个人和商业两类。周老板自己的铺是商业保险，他儿子的茶餐厅也是。但如果他儿子买的是给自己员工看的保障计划，那就又要换一种分法了。"

梁志明把那张报表仔细看了一遍。他第一次意识到，同一个客人的同一间店铺，在公司内部可以被贴上不同的标签，走不同的流程，而这些标签加在一起，才构成了整个保险市场的真实面貌。

## 原文定义

保险公司可按内部管理及营运需要自由地将业务分类，常见分类方式包括：按业务类型（英式分为寿险、海上保险、火险及意外险；美式将人寿与非人寿分开）；按业务来源（保险代理人、保险经纪、直接公众）；按客户类型（个人保险与商业保险）。

## 对应点

| 故事元素 | 概念对应 |
|---|---|
| 周老板的货在不同阶段走不同部门 | 按业务类型的部门式分类 |
| 英式火险/海险/意外险 vs 美式分法 | 英式与美式分类差异 |
| 从保险行买 vs 经纪安排 vs 直接来单 | 按业务来源分类（代理/经纪/直接） |
| 张小姐的报表区分个人与商业 | 按客户类型分类（个人/商业） |
| 周老板自己和儿子的店铺要分开算 | 分类影响核保与计费方式 |
