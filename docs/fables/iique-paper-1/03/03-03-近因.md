# 3.3 近因（Proximate Cause）

## 故事

冯船长的"顺发号"进船坞那天，上环的街坊都知道他亏了大钱。

事情要从三个月前说起。那趟从广州回来的航程，船舱里进了海水，泡坏了一批货物。货主们围在码头等赔偿，保险行的核赔员也来了，来了两次，结论都一样：不赔。

理由是：海水进舱，是因为船员为了扑灭厨房小火，把消防水管打开，水顺着甲板流进了货舱。消防水管是船员开的，水是船员放的，所以海水进舱不是"海上风险"，是"人为操作"。人为操作不在承保范围。

冯船长听到这个理由，把茶杯摔了。

陈伯后来跟他解释："保险行说的，是海水进舱的直接原因是水管，不是海上风浪。所以他们认为近因是水管，不是海。"

"那海上风浪呢？"冯船长问，"如果不是海上起了风浪，那点火怎么着的？"

陈伯没说话。

后来冯船长找到梁志明，把这件事从头到尾讲了一遍。梁志明听完，把他那份保单翻到"近因"那一页，说："船长，你要问的不是'哪个原因在时间上最早'，而是'哪个原因直接导致了损失'。"

他把保单合上："但问题在于，'直接'和'间接'之间，保险行和货主经常有分歧。上次那个案子，船员开水管灭火，水管开着的时候，海浪又打进来，海水和消防水混在一起，分不清哪部分是海、哪部分是水管。你说这怎么算？"

冯船长说："那我的货就白泡了？"

梁志明站起来，走到窗边，望着维港的船桅，说："不是白泡。是要搞清楚，是谁先动的手。火先着的，船员才开水管；海浪先进来的，才是真正的近因。保险行要看的是：整件事，从头到尾，哪一步是最直接的推力。"

陈伯在旁边补了一句："老话叫'一因还一因'。找准了那一个原因，后面的事情才好说。"

冯船长把茶杯重新拿起来，喝了一口，凉透了。

那天傍晚，他在码头上站了很久，看着"顺发号"被拖进船坞维修。夕阳把海面染成铜色，收音机里放着邓丽君的《何日君再来》。

他想起梁志明说的话：找准那一个原因。

他想，航海这么多年，海上的风浪他不怕。但文件和条款里的弯弯绕绕，比维港的暗涌还难看透。

## 原文定义

近因（Proximate Cause）是指在法律上被认定为直接导致损失的那个原因。它不一定是时间上最早的原因，但一定是直接引发损害的关键环节。保险赔偿取决于近因是否属于受保危险。

## 对应点

| 故事元素 | 概念对应 |
|---|---|
| 保险行认为"水管是直接原因，故不赔" | 近因不一定是时间上最早的原因，而是直接导致损失的原因 |
| 冯船长反驳"不是风浪起的火吗" | 需要分析事件链条：火→船员开水管→海水进舱，哪个才是最直接的推力 |
| 梁志明解释要找"最直接的推力" | 近因原则要求识别法律上认定的直接原因 |
| 船员开水管灭火，海水和消防水混在一起 | 多个原因交织时，需判断哪个是真正的近因 |
