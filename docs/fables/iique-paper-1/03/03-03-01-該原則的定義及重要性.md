# 3.3.1 该原则的定义及重要性

## 故事

冯船长在维港跑了二十多年船，什么风浪都见过。他有个习惯：每逢出事，爱坐在码头边抽根烟，把事情从头到尾想一遍。他说这叫"盘账"——出事有因果，因果不清，账就算不明白。

那年他的货船从澳门回来，遇上台风尾。船舱进了水，部分干货被浸坏了。货主拿着保单找保险行索赔，保险行说：水损不赔，因为保单保的是搁浅和碰撞，不保水浸。

冯船长不服气。他拿来一根粉笔，在码头的水泥地上画了一串：台风→船身倾斜→海水打入舱口→货物被浸。

"台风是起因，"他指着第一个圈，"没有台风，后面都不会发生。但保险行说他们不保台风，只保搁浅和碰撞。"

梁志明被请来做中间人。他看了半天，问冯船长："船长，你觉得哪个是真正的因？"

"台风啰。"冯船长不犹豫，"其他都是台风带的。"

"但保险行说，要看哪个原因直接造成了损失。他们认为海水入舱是直接原因，而海水入舱不在受保范围里。"

冯船长沉默了一会儿，又点了一根烟。

"那我问你，"他抽了两口才开口，"如果当时船撞上了礁石导致船舱破裂进水，那算碰撞还是算水损？"

梁志明愣了一下："那要看碰撞和水损之间的关系。哪个在先，哪个直接导致了货物损失。"

冯船长把烟头在鞋底下碾灭，说了句："所以这个'哪个在先'，才是真正的账。"

梁志明回去跟Mr. Thompson谈了半个钟头。Thompson说：这个案子的关键在于，台风（不保危险）和船舱入水（水损）之间的因果链条，哪个环节才是"主导成因"。如果台风本身不被承保，但它直接引发了一件受保的事故，那情况就不一样了。

冯船长后来胜诉了。不是因为台风本身可保，而是因为台风导致船身剧烈摇晃，舱盖密封老化破裂——最后的损失，可以追溯到碰撞级别的船体损害。

他在码头请梁志明喝了杯奶茶，说："你记不记得我说过，出事有因果？我现在加一句：因果有主有次，分不清主次，就结不了账。"

## 原文定义

一项损失的近因（proximate cause）是该项损失或事件的有效或主导成因。在多项成因中，必须识别出主导成因以决定是否属于承保范围。

## 对应点

| 故事元素 | 概念对应 |
|---|---|
| 冯船长画因果链条 | 分析损失成因的相互作用 |
| 梁志明说"要看哪个直接造成损失" | 近因是主导成因，非任意起因 |
| Thompson解释"台风→受保事故"链 | 近因不必是最近发生的事件 |
| 冯船长胜诉，因船体损害可追溯至碰撞 | 识别有效/主导成因的方法 |
