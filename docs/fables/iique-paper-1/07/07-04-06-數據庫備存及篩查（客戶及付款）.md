# 7.4.06-i 数据库备存及筛查（客户及付款）

## 故事

梁志明那玻璃板底下的名单，是ICAC的赵先生帮他一个字一个字抄下来的。

那天培训结束后，大多数人已经走了，梁志明还留在会议室，把赵先生带来的那份名单翻了三遍。赵先生没催他，就站在门口等。

"这份名单，"赵先生指着第一页，"是政府宪报刊登的名单，加上美国行政命令第一三二二四号上列明的人。全香港的保险机构，都必须对照这份名单核对你的客户。"

"要核多久？"梁志明问。

"你们的客户有多少人？"

"几千人。"

"那就定期查，"赵先生说，"不是每次新客户开户才查，是整个客户群定期全部过一遍。"

梁志明回到永亨保险行，把名单上常见的姓氏写进了玻璃板底下。陈伯看见了，问他这算什么。

"数据库。"梁志明说，"我们没有电脑，但这个名字库放在这里，每次接新单、每次大额付款、每次有人要改名受益人，先查一遍。"

陈伯想了想，说："这不够。"

"为什么？"

"因为名单会变。"陈伯说，"你抄的是今天的名单，明天那个名字可能就加进去了，也可能被划掉了。你这玻璃板底下的是死的，名单是活的。"

梁志明那天晚上想了很久，第二天去找林若兰。

林若兰在英国律师楼打字的时候，学会了一套方法：每一份名单都标注日期，每三个月对照一次原版更新，有增有减，全用红笔在旧版上注明"新增"或"删除"。她帮永亨保险行做了一套手工版的"数据库备存记录"。

"这就是你们的名单管理系统，"她把那一叠纸交给梁志明，"每次更新，都要记录谁更新的、什么时候更新的、基于哪个来源更新的。查的时候不只要查玻璃板底下的字，还要对照最新版本。"

后来有一次，一个在东南亚做生意的老客户要收一笔大额汇款，收款人写的是一个陌生的英文名字。梁志明拿出那份名单，对照最新版本，发现那个英文名字和一个删除名单上的人只差了一个字母。

他写了内部报告，直接送到了财富情报组。

汇款后来被拦截了。梁志明没告诉那个客户为什么——不是因为他不想说，是因为他不能。指引3说得很清楚：一旦向客户透露你在查他，整条线索就断了。

何师奶后来问他："你天天对着那几张纸，不闷吗？"

"闷。"梁志明说，"但如果我哪天不查了，有一笔不该进来的钱进来了，那闷的事情会更大。"

他把更新记录锁在档案柜最里层，和那份一九七九年的指引3放在一起。

## 原文定义

指引3规定，金融机构须备存记录恐怖分子嫌疑人物及指定人士名称及详细资料的数据库，收录政府宪报刊登的名单及美国行政命令第一三二二四号指定名单，并须持续对整个客户群进行筛查；须设有筛查付款指示的措施，确保不会向指定人士付款；筛查结果须记录在案。

## 对应点

| 故事元素 | 概念对应 |
|---|---|
| 赵先生说"整个客户群定期全部过一遍" | 须对全部客户群持续进行全面的筛查 |
| 梁志明把名单写在玻璃板底下作为快速查阅工具 | 数据库须便于职员查阅 |
| 陈伯说"名单是活的，会变" | 数据库须及时更新，保持与最新名单一致 |
| 林若兰帮永亨建立标注日期、定期更新的手工记录系统 | 名单更新须记录来源、时间和更新者 |
| 发现陌生英文名字与删除名单仅差一个字母 | 筛查须关注近似匹配，高度警觉可疑情况 |
| 梁志明拦截汇款后"不能告诉客户为什么" | 发现疑似恐怖分子关联须保密，不得向客户通风报讯 |
| 梁志明说"不查的话，闷的事情会更大" | 持续筛查是防止资金通过保险渠道流入的关键防线 |
